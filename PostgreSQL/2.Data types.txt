UUID-
UUID is much much better than the SERIAL data type when it comes to
'uniquness' across systems as SERIAL data type generates only unique 
values within a single database. 

for downloading the uuid-
Create Extension if not exists "uuid-ossp";
select uuid_generate_v1()- this uses mac address and timestamp and random number
select uuid_generate_v4() -this completly random 
 
Create table uud 
( proid UUID DEFAULT uuid_generate_v1());

changing default value 
 alter table uud 
 alter column proid
 set default uuid_generate_v4(); 
 
 
HSTORE -
it stores data in key-value pairs
the values are just strings only

Create Extension if not exists hstore;
ex-
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name TEXT,
    attributes HSTORE
);

INSERT INTO products (name, attributes) 
VALUES ('Backpack', 
	'
	"color"=>"blue", 
	"size"=>"large"
	');

TO print specfic key-
